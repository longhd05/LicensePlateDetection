## How to Contribute

1. **Fork the repository**

2. **Create a new branch**

   Use a descriptive branch name:

   ```
   feature/<feature-name>
   fix/<bug-name>
   ```

3. **Make your changes**

   * Write clean and readable code
   * Follow the existing project structure
   * Add comments where necessary

4. **Commit your changes**

   Example commit message format:

   ```
   feat: add license plate detection module
   fix: correct bounding box scaling
   docs: update README
   ```

5. **Push your branch**

   ```
   git push origin <branch-name>
   ```

6. **Create a Pull Request**

   * Clearly describe what your change does
   * Reference any related issues
   * Wait for review before merging

## Code Style

* Follow the project's existing coding style
* Keep commits small and focused
* Avoid committing unnecessary files (logs, datasets, etc.)

## Reporting Issues

If you find a bug or want to suggest a feature:

1. Open an **Issue**
2. Provide a clear description
3. Include steps to reproduce the problem if applicable
